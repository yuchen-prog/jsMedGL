## 1. 概述

### 1.1 目标

在已完成的 3D 体绘制基础上，新增两项交互功能，对齐 RadiAnt 等专业医学影像软件的体验：

1. **中键拖拽调窗** — 在 3D 体绘制模式下，按住鼠标中键拖拽可实时调整窗宽窗位
2. **组织预设** — 一键切换骨窗、肺窗、软组织窗等常用预设，同时设置 colormap + WW/WL

### 1.2 当前状态

| 功能 | 状态 | 说明 |
| :--- | :--- | :--- |
| 3D 体绘制渲染 | ✅ 已完成 | Standard/MIP/MinIP/Average compositing |
| Window/Level 支持 | ✅ 已完成 | `TransferFunction` 驱动 opacity LUT |
| WW/WL 从 2D 同步到 3D | ✅ 已完成 | `normalizeWL()` 转换后调用 `setWindowLevel()` |
| 中键拖拽调窗 | ❌ 未实现 | 现有中键无功能 |
| 组织预设 | ❌ 未实现 | 无预设按钮 |

### 1.3 验收标准

| 功能 | 验收标准 |
| :--- | :--- |
| 中键水平拖拽 | 水平拖拽改变窗宽（window ± delta），右侧拖 = 窗宽增大 |
| 中键垂直拖拽 | 垂直拖拽改变窗位（level ± delta），向上拖 = 窗位增大 |
| 预设切换 | 点击预设按钮，colormap + WW/WL 同时生效 |
| 预设列表 | 至少包含：骨窗、肺窗、软组织窗、脑窗 |
| 实时预览 | 拖拽过程中 3D 渲染实时更新，无明显卡顿 |
| 2D/3D 联动 | 3D 模式调窗时，2D 视图同步更新（可选，与现有行为一致） |

## 2. 技术背景

### 2.1 Opacity 曲线原理

体绘制中，窗宽窗位控制 opacity 曲线（opacity transfer function），决定哪些密度范围的体素可见以及有多透明。

**当前公式（`TransferFunction.ts:118-139`）：**

```
lo = level - window / 2
hi = level + window / 2

opacity(intensity) =
  0                              , intensity ≤ lo
  smoothstep(intensity, lo, hi) , lo < intensity < hi
  1                              , intensity ≥ hi
```

`smoothstep` 输出 `[0, 1]`，再做 `× 255` 存入 256 エントリ LUT。

**效果：**

- **窗宽大** → lo~hi 间距宽 → 曲线平缓 → 很多密度都有 opacity → 显示内容多但对比度低
- **窗宽小** → lo~hi 间距窄 → 曲线陡峭 → 只有窄密度段可见 → 对比度高
- **窗位高** → 曲线中心右移 → 只显示高密度（骨头）
- **窗位低** → 曲线中心左移 → 只显示低密度（肺、空气）

**归一化说明：**

- 2D 视图：WW ∈ [1, 500]，WL ∈ [0, 255]（绝对值）
- 3D 渲染器：window ∈ [0, 1]，level ∈ [0, 1]（归一化）
- 转换：`window3d = WW / 255`，`level3d = WL / 255`

### 2.2 Colormap vs Opacity 独立性

```
intensity → window/level → windowed [0,1]
              ↓                         ↓
       ┌──────────────┐         ┌──────────────┐
       │  color LUT   │         │ opacity LUT  │
       │  (u_colorLUT)│         │(u_opacityLUT)│
       │  256×1 RGB8  │         │  256×1 R8    │
       └──────────────┘         └──────────────┘
              ↓                         ↓
         颜色结果                   opacity 结果
```

两者独立，组合出不同的视觉效果。

### 2.3 组织预设数据结构

预设 = `(colormap, window, level)` 三元组：

```typescript
interface TissuePreset {
  label: string;           // 界面显示名称
  window: number;          // 2D 绝对窗宽 (0-255)
  level: number;           // 2D 绝对窗位 (0-255)
  colormap: ColormapName; // 对应色表
}
```

### Step 3 — 中键拖拽调窗交互

**修改文件：** `packages/renderer-3d/src/VolumeRenderView.ts`

**交互逻辑：**

| 操作 | 效果 | 灵敏度 |
| :--- | :--- | :--- |
| 中键按下 | 开始拖拽 | — |
| 水平拖拽 | `window ± dx * sensitivity` | `sensitivity = 0.005`（归一化） |
| 垂直拖拽 | `level ± dy * sensitivity`（反向） | `sensitivity = 0.005`（归一化） |
| 中键松开 | 结束拖拽 | — |

**实现要点：**

1. **捕获 `button === 1`**（中键）

```typescript
// VolumeRenderView.ts — handleMouseDown
private handleMouseDown = (e: MouseEvent): void => {
  e.preventDefault();
  this.drag = {
    active: true,
    button: e.button,  // 0=left, 1=middle, 2=right
    startX: e.clientX,
    startY: e.clientY,
    lastX: e.clientX,
    lastY: e.clientY,
  };
};

// VolumeRenderView.ts — handleMouseMove
private handleMouseMove = (e: MouseEvent): void => {
  if (!this.drag.active) return;

  const dx = e.clientX - this.drag.lastX;
  const dy = e.clientY - this.drag.lastY;
  this.drag.lastX = e.clientX;
  this.drag.lastY = e.clientY;

  const sensitivity = 0.005;

  if (this.drag.button === 1) {
    // Middle button: adjust window/level
    const newWindow = Math.max(0.01, Math.min(1.0, this.config.window + dx * sensitivity));
    const newLevel = Math.max(0.0, Math.min(1.0, this.config.level - dy * sensitivity));
    this.setWindowLevel(newWindow, newLevel);
    this.emitter.emit('windowLevelChange', { window: newWindow, level: newLevel } satisfies WindowLevelEventData);
  } else if (this.drag.button === 0) {
    // Left button: orbit
    this.renderer.getCameraObject().orbit(-dx * sensitivity, -dy * sensitivity);
  } else if (this.drag.button === 2) {
    // Right button: pan
    this.renderer.getCameraObject().pan(-dx, dy);
  }

  this.emitter.emit('cameraChange', { state: this.renderer.getCamera() } satisfies CameraEventData);
  this.scheduleRender();
};
```

2. **防止中键默认行为**（浏览器中键默认会触发平移）

```typescript
this.canvas.addEventListener('mousedown', this.handleMouseDown);
this.canvas.addEventListener('auxclick', e => {
  // 阻止中键的默认浏览器行为
  if (e.button === 1) e.preventDefault();
});
```

3. **拖拽时禁用 2D crosshair**（可选，视 UX 决定）

**验收：**
- [ ] 中键拖拽时 3D 视图实时更新
- [ ] 拖拽方向与窗宽窗位变化方向符合预期（水平=WW，垂直=WL，向右/上=增大）
- [ ] 中键不触发浏览器默认行为（自动滚动）

applyPreset(preset: TissuePreset): void {
  this.setColormap(preset.colormap);
  // 预设使用 2D 绝对值，需转换为 3D 归一化值
  const nl = normalizeWWWL(preset.window, preset.level);
  this.setWindowLevel(nl.window, nl.level);
}

private normalizeWWWL(w: number, l: number): { window: number; level: number } {
  return { window: w / 255, level: l / 255 };
}

### Step 6 — 双向联动（可选增强）

**背景：** 当前 3D 视图调窗只更新 3D 渲染，2D 视图不变。如需双向联动，需让 demo app 感知 3D 视图的窗宽窗位变化。

**实现方案：**

```tsx
// App.tsx — VolumeViewer
function VolumeViewer({ volume, windowLevel, ... }: VolumeViewerProps) {
  const viewRef = useRef<ReturnType<typeof createVolumeRenderView> | null>(null);

  // 监听 3D 视图的 windowLevelChange 事件
  useEffect(() => {
    if (!viewRef.current) return;
    viewRef.current.on('windowLevelChange', (data: unknown) => {
      const { window: w, level: l } = data as { window: number; level: number };
      // 将 3D 归一化值转换回 2D 绝对值
      const ww = Math.round(w * 255);
      const wl = Math.round(l * 255);
      onWindowLevelChange?.({ window: ww, level: wl });
    });
  }, [onWindowLevelChange]);

  // ... rest
}
```

**注意：** 双向联动可能产生事件循环，需要 `useEffect` 中做相等性检查避免死循环：

```typescript
if (windowLevel.window === ww && windowLevel.level === wl) return;
```

**验收：**
- [ ] 3D 模式中键调窗后，2D 视图同步更新
- [ ] 2D 侧栏调窗后，3D 视图同步更新
- [ ] 两者之间无死循环

### Step 7 — 边界情况处理

| 情况 | 处理 |
| :--- | :--- |
| window 接近 0 | `Math.max(0.01, ...)` 防止除零 |
| level 超出 [0,1] | `Math.max(0, Math.min(1, ...))` clamp |
| 拖拽速度过快 | sensitivity 已限制每次更新幅度，不会跳变 |
| 中键和左键同时按 | 以最先按下的为准（button 值唯一） |
| 3D/2D 切换时 | 重置交互状态，`drag.active = false` |
