## 1. 概述

### 1.1 目标

在已完成的 2D 切片渲染（MPR/Oblique MPR）基础上，实现 3D 体绘制（Volume Rendering）功能，提供交互式 raycasting 渲染、多种投影模式和梯度光照。

### 1.2 核心差异化

| 特性                | MRIcroWeb | NiiVue | jsMedgl (目标) |
| :---------------- | :-------- | :----- | :----------- |
| sform/qform 坐标系   | 忽略        | 部分支持   | 完整支持（已实现）    |
| Raycasting 体绘制    | 无         | 有      | 有            |
| MIP/MinIP/Average | 部分        | 有      | 有            |
| 梯度光照              | 无         | 无      | Phong 模型     |
| Oblique MPR       | 无         | 无      | 完整支持（已实现）    |

### 1.3 验收标准

| 指标      | 目标值                                     |
| :------ | :-------------------------------------- |
| 3D 渲染帧率 | ≥ 24 FPS（256×256×128 体素，开启光照）           |
| MIP 帧率  | ≥ 30 FPS（同上）                            |
| 纹理上传时间  | < 2s（10MB NIfTI）                        |
| 坐标系一致性  | 继承 NIfTI parser 的 RAS↔LPS 转换            |
| 浏览器兼容   | Chrome/Firefox/Safari/Edge 最新版本（WebGL2） |

**关键设计点：**

| 项目              | 说明                                                                      |
| :-------------- | :---------------------------------------------------------------------- |
| 数据来源            | 复用 `SliceExtractor.getNormalizedData()` 的归一化 Uint8Array \[0,255]，避免重复计算 |

| VRAM 预估         | `width * height * depth * 1 byte` 需在上传前检查 `gl.getParameter()`           |

### Task #12 — RayBoxIntersector（射线-包围盒求交）

**前置条件：** Task #1 完成

**职责：** 计算射线与轴对齐包围盒（AABB \[0,1]³ in texture space）的交点

**算法：Slab Method**

```typescript
function intersectBox(
  rayOrigin: [number, number, number],
  rayDir: [number, number, number],
  boxMin: [number, number, number] = [0, 0, 0],
  boxMax: [number, number, number] = [1, 1, 1]
): { tNear: number; tFar: number; entry: [number,number,number]; exit: [number,number,number] } | null {
  // 1. 对每个 slab (x, y, z) 计算 t_min 和 t_max
  // 2. tNear = max(all t_min)
  // 3. tFar  = min(all t_max)
  // 4. 如果 tNear > tFar 或 tFar < 0 → 无交点
  // 5. 返回交点坐标
}
```

**边界情况处理：**

| 情况       | 处理                              |
| :------- | :------------------------------ |
| 射线完全在盒外  | 返回 `null`                       |
| 射线起点在盒内  | `tNear = 0`，`entry = rayOrigin` |
| 射线与边/面重合 | 使用 epsilon tolerance 避免浮点误差     |
| 射线完全在盒内  | 返回完整 entry/exit                 |

**验收标准：**

- [ ] 单元测试覆盖：正面入射、背面入射、侧面入射、完全在外、完全在内、射线过原点

### Task #6 — RaycastingFragmentShader（核心光线行进）

**前置条件：** Task #1, #2, #9, #11, #12 完成

**职责：** GPU 端光线步进采样、体素到颜色的转换、front-to-back compositing

**Fragment Shader 核心逻辑：**

```glsl
precision highp float;

uniform sampler3D uVolumeTexture;        // 3D volume
uniform sampler1D uOpacityLUT;          // opacity lookup
uniform sampler1D uColorLUT;            // color lookup
uniform vec3 uRayOrigin;                 // 射线起点（纹理空间）
uniform vec3 uRayDir;                    // 射线方向（已归一化）
uniform float uTNear;                    // 入口距离
uniform float uTFar;                    // 出口距离
uniform float uStepSize;                // 步长
uniform float uWindow;                  // 窗宽（归一化）
uniform float uLevel;                   // 窗位（归一化）
uniform bool uGradientLighting;         // 是否启用光照
uniform vec3 uLightDir;                 // 光照方向
uniform int uCompositingMode;           // 0=standard, 1=MIP, 2=MinIP, 3=Average

const int MAX_STEPS = 512;
const vec3 BOX_MIN = vec3(0.0);
const vec3 BOX_MAX = vec3(1.0);

void main() {
  // 1. 计算射线入口和出口点
  vec3 entryPoint = uRayOrigin + uRayDir * uTNear;
  vec3 exitPoint  = uRayOrigin + uRayDir * uTFar;

  // 2. 计算射线总长度
  float rayLength = distance(entryPoint, exitPoint);
  float numSteps = min(rayLength / uStepSize, float(MAX_STEPS));
  vec3 step = uRayDir * uStepSize;

  // 3. 采样循环
  vec3 currentPos = entryPoint;
  vec4 accumulated = vec4(0.0);
  float maxIntensity = 0.0;
  float minIntensity = 1.0;
  float sumIntensity = 0.0;

  for (int i = 0; i < MAX_STEPS; i++) {
    if (float(i) >= numSteps) break;

    // 采样 volume
    float intensity = texture(uVolumeTexture, currentPos).r; // [0,1]

    // 映射到 [0,1] 范围
    float windowed = (intensity - uLevel) / uWindow + 0.5;
    windowed = clamp(windowed, 0.0, 1.0);

    // 梯度光照（central difference）—— 仅在 standard compositing 模式下启用
    // MIP/MinIP/Average 无需光照：禁用可节省 6 次纹理采样，大幅提升帧率
    vec3 gradient = vec3(0.0);
    if (uGradientLighting && uCompositingMode == 0) {
      float step = 0.002; // texture space epsilon
      float left  = texture(uVolumeTexture, currentPos - vec3(step, 0, 0)).r;
      float right = texture(uVolumeTexture, currentPos + vec3(step, 0, 0)).r;
      float down  = texture(uVolumeTexture, currentPos - vec3(0, step, 0)).r;
      float up    = texture(uVolumeTexture, currentPos + vec3(0, step, 0)).r;
      float back  = texture(uVolumeTexture, currentPos - vec3(0, 0, step)).r;
      float front = texture(uVolumeTexture, currentPos + vec3(0, 0, step)).r;
      gradient = normalize(vec3(right - left, up - down, front - back));
    }

    // Color lookup
    vec3 color = texture(uColorLUT, windowed).rgb;

    // Phong 光照
    if (uGradientLighting && length(gradient) > 0.001) {
      vec3 normal = -normalize(gradient); // 体数据中梯度反向
      float diff = max(dot(normal, uLightDir), 0.0);
      color *= (0.3 + 0.7 * diff); // ambient + diffuse
    }

    // MIP / MinIP / Average 累积
    maxIntensity = max(maxIntensity, intensity);
    minIntensity = min(minIntensity, intensity);
    sumIntensity += intensity;

    // Standard compositing
    float opacity = texture(uOpacityLUT, windowed).r;
    if (uCompositingMode == 0) {
      accumulated.rgb += color * opacity * (1.0 - accumulated.a);
      accumulated.a += opacity * (1.0 - accumulated.a);

      // Early termination
      if (accumulated.a > 0.99) break;
    }

    currentPos += step;
  }

  // 4. 最终输出
  if (uCompositingMode == 1) {
    // MIP
    float windowed = (maxIntensity - uLevel) / uWindow + 0.5;
    gl_FragColor = vec4(texture(uColorLUT, clamp(windowed,0.,1.)).rgb, 1.0);
  } else if (uCompositingMode == 2) {
    // MinIP
    float windowed = (minIntensity - uLevel) / uWindow + 0.5;
    gl_FragColor = vec4(texture(uColorLUT, clamp(windowed,0.,1.)).rgb, 1.0);
  } else if (uCompositingMode == 3) {
    // Average
    float avg = sumIntensity / max(float(numSteps), 1.0);
    float windowed = (avg - uLevel) / uWindow + 0.5;
    gl_FragColor = vec4(texture(uColorLUT, clamp(windowed,0.,1.)).rgb, 1.0);
  } else {
    // Standard
    gl_FragColor = accumulated;
  }

  // 背景为透明
  if (uTNear < 0.0 && uTFar < 0.0) {
    gl_FragColor = vec4(0.0);
  }
}
```

**性能关键参数：**

| 参数                 | 默认值                 | 可调范围      |
| :----------------- | :------------------ | :-------- |
| `stepSize`         | 0.5 (texture space) | 0.25\~1.0 |
| `MAX_STEPS`        | 512                 | —         |
| `gradient epsilon` | 0.002               | —         |

**验收标准：**

- [ ] 标准 compositing 渲染正确显示体数据
- [ ] MIP 模式显示最大强度投影
- [ ] early termination 在 opacity 累积到 0.99 后停止步进

### Task #7 — GradientLightingSystem（梯度光照）

**前置条件：** Task #6 完成（嵌入 shader）

**职责：** Phong 光照模型的实现（在 raycasting shader 内）

**光照模型（简化 Phong）：**

```
I = I_a * k_a + I_l * k_d * max(dot(N, L), 0)
```

- `I_a = 0.1` (ambient)
- `k_d = 0.7` (diffuse)
- `I_l = 1.0` (light intensity)
- `N = -normalize(gradient)` (表面法线 = 负梯度)
- `L = normalize(lightDirection)`

**梯度计算（central difference）：**

```glsl
// 在 texture space 中采样 6 邻域
float left  = texture(uVolumeTexture, pos - eps.x).r;
float right = texture(uVolumeTexture, pos + eps.x).r;
float down  = texture(uVolumeTexture, pos - eps.y).r;
float up    = texture(uVolumeTexture, pos + eps.y).r;
float back  = texture(uVolumeTexture, pos - eps.z).r;
float front = texture(uVolumeTexture, pos + eps.z).r;

vec3 grad = vec3(right - left, up - down, front - back) / (2.0 * step);
```

**光照启用条件：**
- `standard` compositing 模式：`uGradientLighting=true` 时启用
- `mip` / `minip` / `average` 模式：**始终禁用光照**（MIP 取最大/最小值无需法线，启用光照无临床意义且徒增采样开销）

**验收标准：**

- [ ] 光照开启时，体表面对比度增强，有明显的明暗变化
- [ ] 光照关闭时，退化为纯 compositing 效果

### Task #8 — WebGLVolumeRenderer（渲染器主类）

**前置条件：** Task #1, #2, #9, #11, #12, #6, #7, #10 完成

**职责：** 整合所有组件，提供完整的渲染管线

**渲染管线：**

```
1. VolumeCamera.getViewMatrix() + getRayDirection(pixelX, pixelY)
        ↓
2. RayBoxIntersector.intersectBox(rayOrigin, rayDir)
        ↓
3. 更新所有 shader uniforms:
   - uRayOrigin, uRayDir, uTNear, uTFar
   - uVolumeTexture (bind)
   - uOpacityLUT, uColorLUT (bind)
   - uWindow, uLevel, uCompositingMode, uGradientLighting, uLightDir
        ↓
4. gl.drawArrays(gl.TRIANGLE_STRIP, 0, 4)  ← 全屏 quad
        ↓
5. gl.readPixels / canvas.toBlob  ← 截图
```

**Uniform 更新策略：**

- Camera 变化（orbit/pan/zoom）→ 重新计算 ray，触发 render
- Window/Level 变化 → 仅更新 uWindow/uLevel uniform，**不重新上传 volume texture**
- Colormap 变化 → 仅更新 uColorLUT texture
- Volume 数据不变 → texture 缓存

**验收标准：**

- [ ] 首次渲染 < 100ms（不含纹理上传）
- [ ] orbit 交互无卡顿（≥ 24 FPS）
- [ ] `dispose()` 释放所有 WebGL 资源

### Task #13 — OrientationCube（方位指示器）

**前置条件：** Task #5 完成

**职责：** 在视口角落渲染 3D 方位 Cube（线框立方体），显示 L/R、A/P、S/I 六个面的方向标签，跟随相机旋转。

**UI 布局：**

```
┌─────────────────────────────────┐
│                        ┌─────┐ │
│                        │ 3D  │ │
│    [Volume Rendering]  │View │ │
│                        │Cube │ │
│                        └─────┘ │
└─────────────────────────────────┘
         Cube 位于右下角，约 100×100px
```

**技术实现：**

```
┌─ CameraMatrix (viewMatrix) ─┐
│                                │
│  主渲染相机: camera.viewMatrix  │────► raycast shader
│                                │
│  Cube 相机: 固定正交投影          │────► cube shader (线框立方体 + 文字标签)
│  + 从主相机继承 rotation         │     (平移=0, 旋转=主相机rotation, scale=缩放)
│                                │
│  标签渲染: Canvas 2D overlay   │────► 在 cube 正交投影面上绘制 L/R/A/P/S/I
│                                │     文字方向随 cube 同步旋转
└────────────────────────────────┘
```

**Cube 渲染细节：**

| 项目 | 说明 |
| :--- | :--- |
| 几何体 | 正方体线框（12 条边），不填充面 |
| 位置 | 固定在视口右下角（CSS absolute positioning） |
| 大小 | 80×80~120×120px，与主视口解耦 |
| 颜色 | L/R 轴=RGB(1,0,0)，A/P 轴=RGB(0,1,0)，S/I 轴=RGB(0,0,1)（沿坐标轴颜色编码） |
| 旋转 | 从主相机提取 rotation matrix（移除 translation/scale），应用到 cube |
| 标签 | Canvas 2D 在 cube 正交投影面上用 ctx.fillText 绘制 L/R/A/P/S/I 字符，方向随 cube 同步 |

**L/R/A/P/S/I 标签计算：**

```typescript
}
```

**Cube 的两个渲染层：**

1. **线框层（WebGL）** — 12 条边，颜色编码轴向，旋转跟随主相机
2. **标签层（Canvas 2D）** — L/R/A/P/S/I 6 个字符，在 cube 正交投影面上用 ctx.fillText 绘制

**替代方案（简化）：** 如果 Canvas 2D 标签跟随旋转计算过于复杂，可改为：
- 用 WebGL 单独渲染 6 个 Billboard 三角形，每个 billboard 中心渲染一个字符（纹理贴字）

- [ ] L/R（红色）、A/P（绿色）、S/I（蓝色）颜色编码正确
- [ ] Orbit 操作时 cube 同步旋转，无卡顿
- [ ] 位于视口右下角，不遮挡主视图

## 4. 关键技术决策

### 4.1 已确认决策

| 问题 | 决策 | 说明 |
| :--- | :--- | :--- |
| 渲染 API | **纯 WebGL2** | 与 renderer-2d 保持一致，不引入 Three.js |
| 浏览器兼容 | **现代浏览器** | WebGL2（Chrome 56+, Firefox 51+, Safari 15+），无需 fallback |
| 体积类型 | **Axis-aligned volume only** | 不支持 oblique volume，后续按需扩展 |
| 相机初始视角 | **斜 45°** | `theta = Math.PI/4, phi = Math.PI/4`，展示最佳 3D 效果 |
| Colormap 数值 | **参考开源实现** | 参考 3D Slicer / OHIF Viewer 色表数值 |
| MIP 光照 | **无光照** | MIP 模式不启用梯度光照 |

### 4.2 性能优化策略

1. **Early Ray Termination**：opacity 累积到 0.99 时停止步进
2. **Adaptive Step Size**：在低 opacity 区域使用大步长，高 opacity 区域使用小步长
3. **Transfer Function Cache**：window/level 变化时仅更新 1D LUT，不重新上传 volume
4. **Texture Residency**：大体积（> 256³）自动降采样
5. **Frame Throttling**：连续帧合并，仅渲染最新帧

## 5. 风险与缓解

| 风险                        |     影响     |  概率 | 缓解措施                             |
| :------------------------ | :--------: | :-: | :------------------------------- |
| WebGL 2D texture fallback | 需重写 shader |  低  | 目标用户使用现代浏览器，明确标注 WebGL2 要求       |
| 大体积内存溢出                   |   渲染失败/崩溃  |  中  | 上传前检查 VRAM，预估内存；超限自动降采样          |
| Gradient 光照性能差            |  FPS < 24  |  中  | 提供光照开关；评估后考虑预计算方案                |
| Raycasting 步数过多           |    渲染卡顿    |  中  | MAX\_STEPS 限制；adaptive step size |
| OrientationCube 标签跟随旋转     |   实现复杂度高   |  中  | 优先用简化 Canvas 2D 方案；WebGL billboard 标签作为备选 |
| 医学专用色表数值准确性            |  与临床工具差异   |  低  | 参考 DICOM / 3D Slicer 色表数值            |
